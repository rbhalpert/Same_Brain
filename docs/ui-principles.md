# Same Brain — UI Principles

## Design goals
The UI should feel:
- clean
- playful
- modern
- lightweight
- highly readable
- not cluttered
- not overly gamer-styled

## Two surfaces
### Shared screen
Used for:
- lobby
- room code / QR
- prompt display
- reveal
- leaderboard
- final results

### Phone screen
Used for:
- joining
- entering name
- submitting answer
- locked-in waiting state
- personal round result
- final personal result

## Phone UI rule
The phone interface must be extremely simple.

At any moment the player should mainly know:
- what the prompt is
- what to type
- whether they submitted
- whether they scored

## Shared screen rule
The shared screen is the spectacle layer.
It should be optimized for readability and group energy.

## Reveal rule
The reveal is the payoff.
Matched answers should visually cluster in a way that feels satisfying and obvious.

## Accessibility
- large tap targets
- readable typography
- high contrast
- do not rely only on color
- avoid dense layouts
- keep text concise

## Pacing
- do not stall unnecessarily on interstitial screens
- leaderboard should be brief
- answering should feel quick
- reveal should feel fun but not slow
